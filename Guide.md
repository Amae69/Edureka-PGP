# Capstone Project Setup Guide

This guide provides step-by-step instructions on how to set up the infrastructure, configure Jenkins, install essential plugins, and set up your Grafana dashboard for real-time monitoring.

## 1. Local Pre-requisites
1. Ensure your AWS Access Key and Secret Key are handy.
2. Have an SSH key pair created in the AWS console named exactly `ec2-key` (or update the `key_name` in `terraform/variables.tf`).
3. Download the `ec2-key.pem` file to `~/.ssh/` on your local machine and set its permissions using `chmod 400 ~/.ssh/ec2-key.pem`.

## 2. Infrastructure Provisioning
1. Initialize Terraform to download the AWS provider:
   ```bash
   cd terraform
   terraform init
   ```
2. Provision the Initial Servers (Jenkins, App, and Monitoring):
   ```bash
   terraform apply -auto-approve
   ```
3. Take note of the Public IPs generated from the output.

## 3. Configure the Servers Setup (Ansible)
1. Edit the `ansible/inventory.ini` file and replace `<JENKINS_IP>`, `<APP_IP>`, and `<MONITORING_IP>` with the actual public IP addresses from your Terraform output.
2. Run the Ansible playbooks to completely configure each environment:
   ```bash
   ansible-playbook -i ansible/inventory.ini ansible/setup-jenkins.yml
   ansible-playbook -i ansible/inventory.ini ansible/setup-app.yml
   ansible-playbook -i ansible/inventory.ini ansible/setup-monitoring.yml
   ```

## 4. Jenkins Server Configuration
After `setup-jenkins.yml` succeeds, Jenkins will be fully installed and running on port 8080.

### Initial Access:
1. SSH into the Jenkins server: `ssh -i ~/.ssh/ec2-key.pem ubuntu@<JENKINS_IP>`
2. Retrieve the initial admin password:
   ```bash
   sudo cat /var/lib/jenkins/secrets/initialAdminPassword
   ```
3. Open a browser and visit `http://<JENKINS_IP>:8080`.
4. Paste the password and select **Install suggested plugins**.
5. Create your Admin user.

### Required Plugins:
Navigate to **Manage Jenkins -> Plugins -> Available Plugins** and install the following:
1. **Docker Pipeline Plugin**: Necessary for building and pushing the docker images.
2. **AWS Credentials Plugin**: Necessary to store your AWS AK/SK for Terraform.

*Note: Restart Jenkins after installing these plugins.*

### Required Credentials:
Navigate to **Manage Jenkins -> Credentials -> System -> Global credentials**:
1. **GitHub/Git**: Set up a credential if your remote repo requires authentication.
2. **DockerHub (`dockerhub`)**: 
   - Kind: Username with password
   - ID: `dockerhub`
   - Description: DockerHub Credentials
3. **AWS Credentials (`aws-creds`)**:
   - Kind: AWS Credentials
   - ID: `aws-creds`
   - Add your AWS Access Key and Secret Key.

## 5. Jenkins Pipeline Execution
1. Create a new Jenkins item -> Select **Pipeline** -> Name it `Capstone-Pipeline`.
2. Under the Pipeline section, choose **Pipeline script from SCM**.
3. SCM: **Git**
4. Repository URL: (Enter the URL for this repository on your GitHub).
5. Script Path: `Jenkinsfile`
6. Click **Save** and then **Build Now**.

## 6. Configuring the Grafana Dashboard
After running the `setup-monitoring.yml` and deploying the app via the pipeline, Node Exporter will be sending metrics.

1. Access Grafana at `http://<MONITORING_IP>:3000`.
2. The default login is `admin` / `admin` (it will prompt you to change the password).

### Add Prometheus Data Source:
1. In the left sidebar, click **Connections** -> **Data Sources** -> **Add Data Source**.
2. Select **Prometheus**.
3. Under the HTTP section, set the URL to: `http://localhost:9090` (because Prometheus and Grafana are sharing the same instance). 
   - *Note: If they are on different instances, use `http://<PROMETHEUS_IP>:9090`*.
4. Scroll to the bottom and click **Save & Test**. It should return a green success message.

### Import the Dashboard (CPU, Memory, Disk, Network):
1. In the left sidebar, click the **+ (Plus Icon)** -> **Import**.
2. Under "Import via grafana.com", enter the ID: **1860**. (This is the highly-rated, standard Node Exporter Full dashboard).
3. Click **Load**.
4. In the "Prometheus" dropdown at the bottom, select the Prometheus data source you just created.
5. Click **Import**.

You will now be redirected to a robust Grafana dashboard actively visualizing real-time CPU, Memory, Disk usage, and more generated by your App Server.
